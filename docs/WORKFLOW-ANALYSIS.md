# 爆款工坊 — 创作者工作闭环分析

> 视角：资深游戏买量视频创作者
> 审查日期：2026-03-15
> 审查范围：全站 10 个模块页面 + 模块间数据流
> 状态：v2 — 已整合产品负责人反馈

---

## 一、创作者工作闭环定义

### 1.1 真实工作流（非纯 AI 链路）

```
看到爆款 → 拆解套路 → 产生灵感（Excel创意表）
→ 匹配模版 → 写脚本 → 画分镜 → 生成/制作视频（AI + AE/UE 混合）
→ 质量审核 → 强制命名 → 下载素材 → 上传投放平台
→ 投放中 → 数据回收（按命名精确匹配）→ 迭代优化（回到第一步）
```

关键认知：**视频产出不一定全 AI**。实际生产中，AI 产出的片段常需要和 AE/UE 传统流程的内容做拼接混剪。

### 1.2 闭环映射到模块

| 闭环阶段 | 对应模块 | 路由 | 连通状态 |
|---------|---------|------|---------|
| 1. 看到爆款 | 爆款分析 → 视频库/竞品素材 | `/analysis` | ✅ 有 |
| 2. 拆解套路 | 爆款分析 → 爆款拆解 | `/analysis` | ✅ 有 |
| 3. 产生灵感 | 灵感收集（对应 Excel 创意表） | `/inspirations` | ✅ 有 |
| 4. 匹配模版 | 爆款模版 → 智能匹配 | `/templates` | ✅ 有 |
| 5. 写脚本 | AI 创作 Studio | `/studio` | ✅ 有 |
| 6. 画分镜 | AI 创作 Studio | `/studio` | ✅ 有 |
| 7. 生成视频 | AI 创作 Studio + 传统制作 | `/studio` | 🔶 缺混合流程 |
| 8. 质量审核 | 智能审核 | `/review` | ✅ 有 |
| 9. 强制命名+下载 | 投放素材 | `/materials` | ❌ 缺命名规则 |
| 10. 投放上线 | 投放素材 → 投放信息 | `/materials/[id]` | 🔶 半通 |
| 11. 数据回收 | 数据洞察（按命名精确匹配） | `/weekly-reports` | 🔶 半通 |
| 12. 迭代优化 | 爆款分析 + 知识库 | `/analysis` `/knowledge` | ❌ 断裂 |

## 二、逐模块体验记录 + 问题发现

### 2.1 灵感收集 `/inspirations`

**我在干嘛**：刷抖音/TikTok 看到一个创意想法，想快速记下来。实际业务中，创意是写在 Excel/钉钉在线文档里的。

**现有能力**：
- ✅ 快速记录文字灵感
- ✅ 标题 + 内容 + 标签
- ✅ 支持语音/拍照/截图（底部三个图标）
- ✅ 搜索 + 标签筛选

**缺失功能**：
| 编号 | 缺失项 | 优先级 | 说明 |
|------|--------|--------|------|
| I-1 | 灵感 → 模版匹配 → Studio | 🔴 P0 | 灵感记了但没有出口。流程：灵感 → 先走模版匹配 → 再进 Studio |
| I-2 | 灵感编辑 | 🔴 P0 | 卡片只能删除，不能编辑修改 |
| I-3 | 灵感附加参考视频链接 | 🟡 P1 | 创作者通常是看到某个竞品视频产生灵感，需要贴上原始链接 |
| I-4 | 灵感状态流转 | 🟡 P1 | 缺少"未处理/已用于创作/已归档"状态，不知道哪些灵感用过了 |
| I-5 | Excel 创意表格式导入 | 🟡 P1 | 支持从钉钉/Excel 批量导入创意条目（用户会提供模版） |
| I-6 | 灵感关联素材 | 🟢 P2 | 灵感最终产出了哪个素材？需要反向关联 |

## 三、闭环断裂分析

### 3.1 最严重的断裂点

```
❌ 灵感 ──X──→ 模版匹配 → Studio
   灵感记了但没有出口，用户记完灵感不知道下一步去哪

❌ 素材完成 ──X──→ 强制命名 → 下载
   没有命名规则约束，导致后续数据回收无法匹配

❌ 数据洞察 ──X──→ 素材库/模版/知识库
   投放数据回收了但没有反标到素材，也没有驱动模版优化

❌ Studio ──X──→ 非 AI 内容混编
   AI 视频无法和 AE/UE 传统制作的片段混合，限制了实际使用

❌ 知识库 ──X──→ 专家经验采集
   知识沉淀只能上传文本，无法通过访谈自然地采集专家经验
```

### 3.2 理想闭环 vs 当前断裂

```
理想闭环（含命名约束）:
灵感(Excel) → [匹配模版] → Studio(AI+AE/UE混编) → [审核]
  → 素材库 → [强制命名] → [下载] → 上传投放平台
  → [投放中:回填命名] → 数据回收 → [按命名精确反标]
  ↑                                                │
  └─── [效果好→提取模版] ← [知识沉淀(AI访谈)] ←───┘

当前断裂:
灵感 → (手动复制) → Studio(纯AI) → (手动跳转) → 素材库 → (无命名/无下载) → (手动录入) → 数据回收 → (无反标)
  ↑                                                                                            │
  └──── (无连通) ←─────────────────────────────────────────────────────────────────────────────────┘
```

## 四、方向性决策（已确认）

| 问题 | 决策 | 说明 |
|------|------|------|
| 灵感 → Studio 的传递方式 | **先走模版匹配再进 Studio** | 实际业务中创意写在 Excel 表格里，灵感到 Studio 的衔接走模版匹配。用户会提供 Excel 创意模版 |
| 数据反标的匹配策略 | **精确匹配** | 素材完成后强制命名，投放时确认命名，回收数据时按命名精确匹配。命名是全链路的锚点 |
| 审核通过后的流转 | **审核通过 = 待投放**，改为投放中需人工回填投放命名确认 | 待投放→投放中这一步必须人工操作，确保命名正确 |
| 竞品视频分析 | **上传视频做多模态分析** | 不集成下载工具，用户自行下载后上传视频文件，AI 做视觉+音频+文案全维度分析 |

## 八、专项设计：AI 访谈式知识采集

### 8.1 产品定位

让有经验的创作者/投放老手，通过**聊天对话**的方式，无负担地沉淀经验。AI 扮演访谈者，引导提问，最终产出结构化的 Markdown 知识条目。

### 8.2 用户角色

- **管理员**：创建访谈主题，邀请专家
- **受邀专家**：通过专属链接进入访谈页面（可以是外部用户，不需要登录主系统）
- **知识库**：接收访谈产出的 Markdown

### 8.3 核心流程

```
管理员创建访谈
  ├── 设定主题（如"SLG 买量视频前3秒钩子经验"）
  ├── 设定引导问题清单
  └── 生成专属访谈链接

专家进入访谈页面
  ├── 独立页面，无需登录主系统
  ├── AI 访谈者开场 → 引导问题 → 追问细节 → 举例请求
  ├── 支持语音输入（转文字）
  └── 对话过程实时保存

访谈结束
  ├── AI 整理对话 → 生成结构化 Markdown
  ├── 专家确认/修改
  └── 提交到知识库

知识库展示
  ├── 知识条目标记贡献人
  ├── 贡献排行榜（激励机制）
  └── 知识条目关联审核维度/模版
```

### 8.4 技术方案

- 独立路由：`/knowledge/interview/[token]`（token 鉴权，不需要完整登录）
- AI 后端：DeepSeek 对话模式 + system prompt 定义访谈角色
- 产出格式：Markdown → 直接入库 `knowledge_entries` 表
- 贡献人字段：`contributor_name`, `contributor_role`, `interview_id`

### 8.5 激励设计

- 知识库首页显示"贡献排行"
- 每个知识条目标注"贡献者: xxx"
- 可选：积分奖励（与现有积分系统打通）

## 九、素材命名规则设计（已确认）

> 来源：团队实际使用的命名规范截图 + 渠道明细报表验证

### 9.1 命名的战略意义

命名是全链路的**锚点**：
- Studio 产出 → 按规则命名 → 下载 → 上传投放平台 → 投放数据回收 → 按命名精确匹配回素材库

如果命名不统一，数据回收就无法自动匹配，整个闭环断裂。

### 9.2 命名规则格式（11 标签，下划线分隔）

```
#创建时间_1素材来源_2设计师_3创意_4素材类型_5方向1_6方向2_7代言人_8核心内容_9命名_a配音_b产品_c版式
```

#### 各字段定义

| 序号 | 字段 | 值说明 | 必填 |
|------|------|--------|------|
| # | 创建时间 | 6位日期 YYMMDD，如 `250908` | ✅ |
| 1 | 素材来源 | `Z`=内部自制, `W`=外包(带外包商名), `WZ`=外包+自制, `Y`=亿创, `YZ`=亿创+自制, `M`=美术中心, `P`=平台回收, `O`=omni | ✅ |
| 2 | 设计师 | 必包含≥3人，如 `2福`, `2茹任福` | ✅ |
| 3 | 创意 | 创意策划人 | ✅ |
| 4 | 素材类型 | `AE` 或 `UE` | ✅ |
| 5 | 方向1 | 外包素材填 `W`+内容, 自制素材填 `/` 或具体方向 | 条件必填 |
| 6 | 方向2 | 自制素材填 `Z`+内容, 外包素材填 `/` 或具体方向 | 条件必填 |
| 7 | 代言人 | 非必填项 | ❌ |
| 8 | 核心内容 | 字母代码: `A`=原片, `B`=深度迭代混剪, `C`=3人混剪迭代, `D`=玩法解说混剪, `E`=扒片头大字报, `F`=镜像改色扒, 对应不同消耗系数 | ✅ |
| 9 | 命名 | 含序号+自定义命名，如 `S997宗主登基建造`, `G11空中扩张2` | ✅ |
| a | 配音 | `A`=无配音, `B`=普通话电子音, `C`=普通话真人, `D`=台语电子音, `E`=台腔真人, `F`=其他方言电子音, `G`=其他方言真人, `H`=混合 | ✅ |
| b | 产品 | 产品名，如 `造化`, `三冰` | ✅ |
| c | 版式 | `横`, `竖`, `方` | ✅ |

#### 实际素材名称示例（从渠道报表提取）

#250108_1Z_2福_3福_4AE_5剧_6IP_8场景_9213一念剧情迭1_aC_b造化_c竖
#250917_1W鲸游_2慰_3慰_4AE_5W【副玩】【地块合成】_8A原片_9G11空中扩张2_aA_b造化_c竖

### 9.3 系统强制约束流程

```

  ↓

### 10.1 UE 脚本模版（多 Sheet 工作流）

UE 脚本是完整的**外包协作文档**，包含多个 Sheet：

|-------|------|--------|

| 资产确认 | 外包商资产确认 | 资产名/截图/备注/需求方反馈 |
| 分镜反馈 | 分镜审核 | 镜号/画面/描述/备注/修改意见/参考示例 |
| T-pose节奏确认 | 动作节奏审核 | 问题截图/反馈描述/示例参考/是否完成修改 |
| layout反馈 | 布局审核 | 同上 |
| 成片反馈-v1/v2 | 成片迭代审核 | 同上 |

```
基础信息:
  脚本名称: 【一级标签】【二级标签】自定义命名-创意人员名
