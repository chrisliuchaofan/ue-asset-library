FOCUSED SPRINT PLAN · 2026-05-09

爆款工坊专项优化开发计划

本计划用于重启主项目 资产库web，不继续 lite 版，不重构部署，不追求大而全。第一阶段只验证一个核心命题： 爆款模版能否被复刻，并被真实消耗数据验证。

主线：爆款拆解 → 模版 → 复刻 → 数据反馈 范围：主项目内开发 策略：先闭环，再扩展 协作：主 Agent + 计划 / 开发 / 验收

一、最终聚焦

第一阶段不做“完整爆款工坊”，只做 “验证一个爆款模版是否值得继续复刻”。

1. 已验证爆款 从内部高消耗素材或竞品素材中选出母版候选。
2. 提炼模版 固定输出 5 要素，而不是泛泛的 AI 分析。
3. 绑定复刻 把同一公式下的新素材绑定为复刻子素材。
4. 回收数据 通过周报或 TD 导出回写消耗、点击、成本等指标。
5. 判断有效 模版进入有效、观察中或暂停，反哺下一轮创作。
判断标准不是“页面看起来完整”，而是能不能跑通一条黄金路径：选爆款素材 → 提炼模版 → 绑定复刻素材 → 上传数据 → 模版页显示复刻效果。

二、为什么要这样减法

之前的风险

计划同时包含爆款池、Studio、TD 接入、驾驶舱和复杂趋势，容易重新发散成五个半成品。

现在的取舍

TD 实时同步先不做，完整 ROI 大盘先不做。先让主项目把反馈数据吃进去，并聚合到模版上。

比赛最强叙事

不是“AI 能生成素材”，而是“AI 提炼的公式能被复刻，并由投放数据验证”。

三、第一阶段只做四件事

优先级 任务 要解决的问题 验收标准
P0 补齐投放指标字段映射 主项目已有字段，但周报 / TD 导出里的展示、点击、CTR、CPC、CPM、新增成本、付费成本等不一定都能稳定解析和回写。 上传一份数据后，素材详情能看到消耗、展示、点击、CTR、CPC、CPM、新增成本、新付费数、新付费成本。
P0 建立模版 - 素材关系 当前 `sourceMaterialIds` 只能表达来源素材，不能表达母版、复刻、竞品参考的关系。 一个模版能绑定三类素材：来源爆款、复刻素材、竞品参考；复刻素材只能清楚地归属于某个模版。
P1 做模版效果详情页 需要一个能给评委和团队一眼看懂的页面：这个模版来自哪里，复刻了什么，结果怎样。 页面显示来源爆款、模版 5 要素、复刻素材列表、累计消耗、最好复刻素材、当前判断。
P1 跑通黄金演示链路 避免“页面存在但业务不通”。 从素材库选一条高消耗素材，到模版页看到复刻素材消耗反馈，全链路可演示。

四、数据设计

1. 指标字段

  • 消耗：`consumption`
  • 展示：`impressions`
  • 点击：`clicks`
  • 点击率：`ctr`
  • 点击成本：`cpc`
  • 千次展示成本：`cpm`
  • 新增成本：`newUserCost`
  • 新付费数：`firstDayPayCount`
  • 新付费成本：`firstDayPayCost`
  • ROI：`roi`，第一阶段可选

2. 匹配优先级

  1. `platformId` 对 TD 的视频 md5 / 素材 ID / 创意 ID。
  2. `platformName` 对投放平台素材命名。
  3. `materialNaming` 对系统标准命名。
  4. `name` 做兜底模糊匹配。

第一阶段不直接接 TD API，只保证 Excel / 导出数据能稳定进入主项目。

新增关系模型 说明 第一阶段字段
template_material_relations 用于表达一个模版和多个素材之间的关系。 `template_id`、`material_id`、`relation_type`、`note`、`created_at`
source 来源爆款素材，说明这个模版从哪里提炼。 用于展示“母版来源”。
replica 复刻素材，说明这个模版后来产出了什么。 用于聚合复刻素材数据。
competitor_reference 竞品参考素材，说明外部爆点依据。 用于竞品拆解，不要求有真实投放回写。

五、模版 5 要素

AI 拆解不能只输出一段分析，必须沉淀成稳定结构。模版详情页只认这 5 个字段：

核心结构

这条素材为什么成立，前中后段怎么组织。

开头钩子

前 3 秒靠什么抓住用户。

爽点 / 付费点

用户继续看或愿意点击的核心刺激。

可变变量

复刻时可以替换的人设、场景、文案、资产。

不能改的部分

一改就失效的节奏、冲突、镜头或承诺。

判断结果

基于复刻素材表现，标记有效、观察中、暂停。

六、多 Agent 协作模式

Main Agent

主 Agent：统筹，不开放

  • 只做任务派发、范围控制、合并判断。
  • 不直接沉迷写代码，避免视角被局部实现带偏。
  • 每个阶段只问：是否服务“模版被数据验证”这件事。
Planner

计划 Agent

  • 拆今日任务。
  • 明确今天不做什么。
  • 检查数据模型和接口边界。
Developer

开发 Agent

  • 只改主项目。
  • 按优先级完成字段映射、关系表、API、页面。
  • 不碰 lite 项目和部署结构。
Acceptance

验收 Agent

  • 跑类型检查、构建和关键路径。
  • 检查数据是否真实写回。
  • 输出可演示、不可演示、风险点。

七、开发顺序

顺序 开发内容 验收动作 是否可延期
1 补周报 / 导出字段解析与回写。 上传样例数据，检查素材表指标是否更新。 不可延期
2 新增模版 - 素材关系表和服务函数。 能把一个素材绑定为来源，另一个素材绑定为复刻。 不可延期
3 给模版详情页聚合复刻素材数据。 页面显示累计消耗、最好复刻素材、最近数据时间。 尽量完成
4 从素材详情或素材库增加“提炼为模版 / 绑定到模版”。 不进数据库后台也能完成一次演示操作。 尽量完成
5 修演示链路和文案。 录屏或现场演示时不需要解释太多。 可压缩

八、明确不做

第一阶段不做

  • 不做 TD 实时自动同步。
  • 不做完整 ROI 驾驶舱。
  • 不做新的 lite 项目。
  • 不做全量竞品爬取。
  • 不做复杂多维趋势图。
  • 不重构主项目导航和部署。

只保留的演示重点

  • 一条爆款素材能成为模版。
  • 一个模版能绑定复刻素材。
  • 复刻素材能接收数据反馈。
  • 模版页能判断这个公式是否有效。

九、验收清单

检查项 通过标准
数据回写 素材表至少能写入消耗、展示、点击、CTR、CPC、CPM、新增成本、新付费数、新付费成本。
关系表达 一个模版能同时拥有来源素材和复刻素材，不靠人工记忆。
模版效果 模版详情页能看到复刻素材表现，并给出有效 / 观察中 / 暂停判断。
黄金路径 从素材库进入，到模版页看到复刻效果，全程不需要切到 lite 项目。
技术验证 类型检查、构建、关键页面手动验证通过。

十、第一刀

建议立刻开工的不是页面，而是两个根基： 投放指标字段映射 和 模版 - 素材关系表。

这两个稳了，后面的模版效果页、素材库入口、Studio 复刻入口都会沿着正确的数据结构生长，不会再变成一组互相看起来相关但业务上断开的页面。
