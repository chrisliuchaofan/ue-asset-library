# 爆款工坊重启报告

> 生成日期：2026-05-09  
> 盘点范围：`资产库web` 主应用、现有产品文档与开发计划  
> 结论：项目不是从零重启，而是进入“收束和减法”阶段。

---

## 1. 一句话结论

`资产库web` 的 V1/V2 主干已经能构建，AI 流水线也不是坏掉的半成品；真正的问题是 V3 之后曾同时长出两条方向：

1. **产品闭环线**：灵感、模版、Studio、审核、命名、下载、周报反标。
2. **比赛演示线**：竞品复刻、裂变矩阵、ROI 故事化驾驶舱。

现在已经决定清理比赛演示线。重启建议：**主产品只保留闭环线，不再维护独立比赛样机，也不要把样机思路回流进主应用信息架构。**

## 4. 还剩多少没做完

按“代码是否存在”算，V3 没有大片空白；按“能否交给真实用户跑完整闭环”算，还差关键 4 件：

1. **素材命名与投放信息保存链路**  
   更新 schema/API，让命名、命名字段、投放平台名、计划 ID、账户、上线日期真正写入材料表。

2. **命名强制约束**  
   未命名不能进入审核通过/已投放；下载必须使用标准命名；从已通过到已投放时必须确认平台素材名。

3. **周报数据反标真实回写**  
   周报上传后不仅要显示匹配结果，还要把消耗、展示、点击、CTR、CPC、CPM、新增成本等写回素材库。

4. **一个基础 ROI/消耗看板**  
   不需要大而全，先做 TOP20 消耗排行、首日付费成本排行、未匹配素材列表。没有这一步，“数据闭环”讲不出业务价值。

粗略估算：  
**主应用离可演示闭环还差 3-5 天；离可给真实团队稳定使用还差 1-2 周。**

## 5. 建议立刻做减法

### 先保留

| 保留项 | 原因 |
| --- | --- |
| 灵感 -> 模版 -> Studio | 已接近完成，是创作入口 |
| Studio 脚本/分镜/视频/上传替换 | 已经是核心生产台 |
| 审核中心 | 已有知识库动态维度，是产品壁垒 |
| 素材命名 + 下载 | 是数据回收锚点，必须补完 |
| 周报匹配 + 素材反标 | 是“AI 帮我提消耗”的证据 |
| 基础排行榜看板 | 用最小 UI 讲清 ROI 故事 |

### 暂时砍掉或隐藏

| 砍掉/隐藏项 | 原因 |
| --- | --- |
| 完整视频拼接、BGM、字幕系统 | 复杂度高，不是当前闭环断点 |
| 广告平台 API 自动对接 | 先用 Excel，平台 API 会拖慢重启 |
| UE 外包协作全流程 | 流程重，适合二期 |
| 素材到期/淘汰管理 | 不是当前成败关键 |
| 知识自动沉淀、贡献排行 | AI 访谈已经够高级，先不要继续扩张 |
| 模版收藏、资产收藏、搜索增强 | 锦上添花 |
| Landing Page 大改 | 当前不缺包装页，缺闭环可信度 |
| 比赛演示线 / 独立样机 | 已清理，不再作为主应用路线；后续不要把样机信息架构回流进主产品 |

## 6. 两条路线怎么处理

### 路线 A：主产品重启

目标：让创作者真的能跑一条链路。

```
灵感/竞品 -> 模版匹配 -> Studio 生成/混编 -> 审核 -> 标准命名 -> 下载投放 -> 周报反标 -> 看排行
```

这条线建议作为接下来唯一主线。

### 路线 B：比赛演示

该路线已废弃并清理，不再维护独立演示样机。历史上的演示链路是：

```
上传竞品素材 -> AI 拆解 -> 裂变多版脚本 -> 导出 AE/JSON -> 用 Mock ROI 讲业务故事
```

后续不再把这条线作为独立入口，也不把它直接并进 `资产库web`。主应用只保留真实产品闭环中必要的灵感、模版、Studio、审核、命名、周报反标与看板能力。

## 7. 推荐第一周重启计划

### 2026-05-09 已开工记录

- 已扩展 `MaterialUpdateSchema`：素材更新接口现在可接收状态、投放字段、标准命名字段、投放指标字段。
- 已给素材状态流转加闸门：未完成标准命名不能标记为已通过/已投放；未回填平台素材名不能标记为已投放。
- 已补状态失败提示：素材详情页会显示命名/投放信息缺失原因，不再静默失败。
- 已接上周报反标：周报后台匹配成功后，会调用 `writebackMetrics` 把消耗、展示、点击、CTR、CPC 等指标写回素材库。
- 已验证：`npm run typecheck` 通过，`npm run build` 通过。

### Day 1：修管道

- 扩展 `MaterialUpdateSchema`，放行 V2/V3 字段。
- 验证素材命名、投放信息、消耗指标是否能真实写入 Supabase。
- 清掉主应用里的临时日志文件，确认当前 Git 状态。

### Day 2：命名闭环

- 未命名素材禁止标记为已通过/已投放。
- 下载文件名强制使用 `materialNaming`。
- 已通过 -> 已投放时，弹出或内嵌确认平台素材名。

### Day 3：周报反标

- 周报上传后调用 `writebackMetrics`。
- 在报告页显示成功/失败/未匹配列表。
- 素材卡片和详情页显示最新消耗指标。

### Day 4：最小看板

- 做 TOP20 消耗排行。
- 做 TOP20 首日付费成本排行。
- 做未匹配素材清单。

### Day 5：一条完整演示

- 用 1 条真实或模拟素材跑完整链路。
- 记录演示脚本：从灵感到投放数据回收。
- 用主应用闭环完成演示，不再准备独立答辩入口。

## 8. 当前最该避免的坑

1. **不要再新增大模块**  
   现在已经不缺模块，缺的是几个关键动作是否真的闭环。

2. **不要被“页面存在”误判为“功能完成”**  
   这次盘点里最大的问题就是若干页面/组件已经有了，但保存、回写、强制校验还没完全打通。

3. **不要重新混合主产品和比赛样机**  
   独立样机已清理。后续只保留能支撑真实产品闭环的功能，不再为短期演示单独扩出一条信息架构。

4. **不要追求全自动出片**  
   这个项目最可信的定位仍然是“AI 副驾驶 + 数据闭环”，不是承诺一键生成完美广告。

---

## 9. 建议重启口号

接下来不要叫“继续开发 V3 全量功能”，建议改成：
