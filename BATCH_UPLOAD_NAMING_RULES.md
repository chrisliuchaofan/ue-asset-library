# 批量上传文件命名规则

## 推荐命名规则

为了确保批量上传时能正确识别和抓取同命名的所有媒体文件，请遵循以下命名规则：

### 规则1: 直接匹配（最简单）

```
资产名称: 251010_支_034卧龙凤雏_三冰_方

文件名:
- 251010_支_034卧龙凤雏_三冰_方.mp4
- 251010_支_034卧龙凤雏_三冰_方.png
- 251010_支_034卧龙凤雏_三冰_方-image.png
```

**说明**: 文件名直接包含资产名称，系统会自动识别。

### 规则2: 带前缀数字

```
资产名称: 251010_支_034卧龙凤雏_三冰_方

文件名:
- 3-251010_支_034卧龙凤雏_三冰_方.mp4
- 3-251010_支_034卧龙凤雏_三冰_方.png
- 3-251010_支_034卧龙凤雏_三冰_方-image.png
```

**说明**: 系统会自动去除前缀数字（如 `3-`）进行匹配。

### 规则3: 带后缀标识

```
资产名称: 251010_支_034卧龙凤雏_三冰_方

文件名:
- 251010_支_034卧龙凤雏_三冰_方-image.mp4
- 251010_支_034卧龙凤雏_三冰_方-image.png
- 251010_支_034卧龙凤雏_三冰_方-image2.png
- 251010_支_034卧龙凤雏_三冰_方-preview.png
- 251010_支_034卧龙凤雏_三冰_方-thumb.png
```

**说明**: 系统会自动去除 `-image`、`-image2`、`-preview`、`-thumb` 等后缀进行匹配。

### 规则4: 组合使用

```
资产名称: 251010_支_034卧龙凤雏_三冰_方

文件名:
- 3-251010_支_034卧龙凤雏_三冰_方-image.mp4
- 3-251010_支_034卧龙凤雏_三冰_方-image.png
- 3-251010_支_034卧龙凤雏_三冰_方-image2.png
```

**说明**: 系统会同时去除前缀数字和后缀标识进行匹配。

## 匹配逻辑

系统使用以下策略进行文件匹配：

1. **完全匹配**: 文件名关键部分完全匹配资产名称
2. **包含匹配**: 文件名包含资产名称的关键部分
3. **双向匹配**: 资产名称包含文件名的关键部分
4. **直接匹配**: 文件名去除扩展名后直接包含资产名称

## 支持的媒体格式

### 图片格式
- `.jpg` / `.jpeg`
- `.png`
- `.gif`
- `.webp`
- `.bmp`
- `.svg`

### 视频格式
- `.mp4`
- `.webm`
- `.mov`
- `.avi`
- `.mkv`

## 示例

### 示例1: 标准命名

**CSV中的资产名称**: `building_047_FGH_副楼`

**ZIP中的文件**:
- `building_047_FGH_副楼.png` ✅ 会被识别
- `building_047_FGH_副楼.mp4` ✅ 会被识别
- `building_047_FGH_副楼-image.png` ✅ 会被识别

### 示例2: 带前缀

**CSV中的资产名称**: `251010_支_034卧龙凤雏_三冰_方`

**ZIP中的文件**:
- `3-251010_支_034卧龙凤雏_三冰_方-image.mp4` ✅ 会被识别
- `3-251010_支_034卧龙凤雏_三冰_方-image.png` ✅ 会被识别
- `3-251010_支_034卧龙凤雏_三冰_方-image2.png` ✅ 会被识别

### 示例3: 不匹配的情况

**CSV中的资产名称**: `building_047_FGH_副楼`

**ZIP中的文件**:
- `building_046_FGH_城墙.png` ❌ 不会被识别（名称不同）
- `other_file.png` ❌ 不会被识别（不包含资产名称）

## 提示

1. **在CSV中填写文件名**: 如果自动匹配失败，可以在CSV的"预览图和视频"列中显式填写文件名（逗号分隔）
2. **查看控制台**: 如果文件没有被识别，查看浏览器控制台的警告信息，了解匹配失败的原因
3. **统一命名**: 建议在同一资产的所有文件中使用统一的命名规则
